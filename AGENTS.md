# 仓库协作说明

## 文档同步规则

- 根目录的 `README.md`（中文，默认）、`README_EN.md`、`README_JA.md` 是同一份项目说明的多语言版本。
- 修改其中任意一个 README 时，必须同步检查并更新其他语言版本，确保结构、关键信息和对外表述保持一致。
- 涉及仓库定位、项目关系、对外说明等信息变更时，不允许只修改单一语言版本。
- `docs/` 目录默认用于正式文档；在当前 Web 化改造阶段，如旧桌面端文档已失效，可以直接清空，不保留历史图片和旧引用。
- `docs-dev/` 目录用于开发计划、迁移方案、执行记录和阶段性设计文档；新的改造计划优先写入 `docs-dev/`。

## 语言规则

- 仓库内默认使用简体中文沟通与说明。
- 多语言 README 可以分别使用对应语言编写，但语义必须保持一致。
